# Build plan — v0.1.1 (UX polish + prompt engineering)

A patch-version cycle between MVP (0.1.0, shipped 2026-04-29) and the next feature wave. **Two parallel work streams:** (a) mobile-side UX polish — the rough edges that became visible once the app was usable end-to-end on TestFlight; (b) prompt-engineering tranche — addresses gaps revealed by the first real-world test calls (see "Prompt engineering tranche" section below). Both ship together as 0.1.1.

The UX cycle splits into two passes: a **visual-identity + core surfaces** pass (the priority items below — splash, auth, onboarding, home, call, plugin shell), and a **rough-edges** pass (the smaller backlog-derived items pulled forward into this patch).

Items below are copied from `backlog.md` so this doc is self-contained for the cycle. When an item ships, mark it ✅ here AND remove (or strikethrough + mark "shipped in 0.1.1") in the backlog. If we change our minds on scope mid-cycle, items can move back to `backlog.md` freely.

**Legend**

- ✅ — done
- ✴️ — partial / in progress
- ⏺️ — open / not started
- ⛔ — blocked

The first real-world test calls on 2026-05-01/02 surfaced concrete gaps these items address: Pro fan-out commits zero sections on perfectly-substantive transcripts because the prompt is too conservative about creating sections, and there's no claim-level audit trail to debug *why* each decision was made.

#### Bundle 1 — wiki structure is user-driven, not model-inferred (2026-05-06)

A coherent four-layer fix triggered by the 2026-05-06 incident (sub-topics of "Consensus" filed under an invented `Concepts` top-level bucket instead of nesting under their parent project). Promoted from item 4 ("Refactor handling of page types") with priority bumped P1→P0 because the user flagged this as blocking MVP-functional status. All four layers shipped together:

- ✴️ **Layer 1 — Seed `projects` bucket + universal sub-page trim (profile + agent-scope).** Added `PROJECT_PAGES` (flat root only) to `apps/server/src/seed/seed.constants.ts`. Trimmed PROFILE_PAGES from 10 (root + 9) to **just the root** (1), AND trimmed AGENT_SCOPE_PAGES from 5 (root + 4) to **just the root** (1). Every `profile/<area>` and `assistant/<area>` sub-page is now created on-demand by ingestion when content matches. The principle: empty seeded sub-pages aren't load-bearing structure; they're noise that pollutes the structural-snapshot preload AND gives the model false impressions of what the user (or assistant) has actually accumulated. Total seeded now **8 wiki_pages**: 1 agent-scope root + 1 profile root + 5 todos (root + 4 status buckets, retained because the status workflow is functional structure not empty containers) + 1 projects root. **Profile vocabulary** (lives in Flash + Pro + Live Agent prompts): the 7 askable areas (`profile/goals`, `/life-history`, `/health`, `/work`, `/interests`, `/relationships`, `/preferences`) plus the 2 emergent (`profile/values`, `/psychology`); non-canonical sub-pages (e.g. `profile/finances`, `/spirituality`) when content warrants. **Agent-scope vocabulary** (lives in `apps/worker/src/ingestion/agent-scope.ts`): 3 canonicals — `assistant/recurring-themes`, `assistant/preferences-noted` (**load-bearing**: rewrote description to make this the operational reference for HOW to engage this specific user — covers both inferred patterns AND explicit user instructions like "be more concise"; complements user-scope `profile/preferences` which is the user-visible stated record), `assistant/open-questions`. Plus 2 non-canonical examples: `assistant/strengths`, `assistant/blind-spots`. **Cut `assistant/observations`** (everything in agent-scope IS an observation — redundant; general observations now land directly on the `assistant` root). **Cut `assistant/growth-edges`** (nudges the agent toward coaching/evaluation, which isn't its role). **Flash prompt** got a new section "Profile sub-pages → propose on-demand when content matches" with the canonical vocabulary, mirroring the existing "todos/todo unconditional flag" pattern. **Agent-scope prompt** "Where observations land" rewritten to reflect on-demand-only. **Pro prompt** wiki-ontology updated. **Live Agent prompt** wiki-structure section trimmed to "use the preload to see what currently exists for THIS user." Existing users won't get the seed change retroactively — see TODO list below.

## Demo target

A returning TestFlight user who used 0.1.0 should immediately notice:

1. The visual shell — LavaLamp backdrop, Comfortaa wordmark, glass tiles, redesigned splash/auth/onboarding — has fully replaced the MVP look.
2. The call screen orb is shader-driven and reacts to call state (idle / user-speaking / agent-speaking / interrupted) instead of the placeholder animation.
3. A generic call's response style plausibly modulates per turn (Conversational Routing Phase 1) — verifiable by reading the transcript across self-exploration / info-seeking / brainstorm / capture turns.
4. Onboarding feels coherent with the four core UX principles (Proactiveness / Transparency / Continuity / Autonomy) — agent doesn't barrel through agenda; stops when user stops; tells user what it's capturing.
5. New content from a real call lands under the right hierarchy — sub-topics nest under their parent project, not under invented `Concepts` / `Places` buckets; explicit user direction ("move A under B") is respected end-to-end.
6. `call_transcripts.pro_fan_out_response` is populated for new calls — claim-level audit trail recoverable for any post-MVP ingestion.

If those six are true, the patch shipped.
