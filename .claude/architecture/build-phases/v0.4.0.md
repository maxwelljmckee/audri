## In scope

### Track A — Foundation

#### A0. Quick win — ingestion traffic director

1. ✅ **Ingestion traffic director — top-of-pipeline routing layer.** Empty-bypass branch shipped 2026-05-19; threshold tuned 3→5 after dogfood. Telemetry live. task_only branch deferred to Todo specialist (#13); settings_only effectively handled by the parallel pre-Pro settings specialist (no fast-path branch needed at v0.4.0 scale). Thin deterministic dispatch BEFORE Flash retrieval and Pro fan-out. Routes each transcript to one of N branches:
    - **Empty bypass** (pure heuristic: word count + ASR confidence + duration) — skip all downstream services.
    - **Task-only fast path** (Flash classifier with structured output) — skip Pro fan-out, route directly to action dispatcher.
    - **Standard path** — current Flash → Pro → commit pipeline.
    
    Bias toward standard under uncertainty (false-negative on task-only = silent information loss; false-positive on standard = wasted inference cost). Telemetry mandatory on every routing decision — informs the deferred Pro-fork question (`backlog.md` "Pro fan-out fork — Notes + Task Dispatch split"). Pairs with the P1 backlog "Application-level wall-clock timeout on Gemini fan-out calls" entry — both should land in this quick-wins batch. See `specs/customization-framework.md` § Quick Wins for the full architectural framing.

#### A1. Prompt decomposition / recomposition

2. ✅ **Layered prompt-composition seam — five named layers.** Live Agent migrated 2026-05-19. `apps/server/src/calls/live-agent-prompt-layers/` houses identity / capability / behavioral (with 6 submodules: style, reasoning, tool-use, wiki-workflow, interview, custom-rules) / contextual / grounding. Snapshot harness verifies byte-equivalence + intentional content changes (10 scenarios). composeSystemPrompt aliased to composeLiveAgentPrompt for backward compat. **Open:** ports to Rumi (ingestion) + Dreams when their refactor lands. Refactor every prompt-emitting site to compose from discrete layers, each with a defined source-of-truth:
    1. **Identity / role** — who the agent is. Static, per-agent.
    2. **Capability advertisement** — what the agent can do. Generated from plugin registry (see A4 App Map).
    3. **Behavioral** — knobs + active NL overlays. Injected per inference.
    4. **Contextual** — per-call / per-page / per-task data. Dynamic, request-specific.
    5. **Grounding** — retrieval results, enrichment lookups, tool outputs. Injected post-retrieval.
    
    Each layer has exactly one injection point and one source of truth. **Implementation discipline:** (a) build for Live Agent end-to-end first (most prompt surfaces, hardest semantics) — refactor cleanly, prove the seam holds, then port Rumi + Dreams; (b) behavioral test pre-refactor — capture 3-5 real input/output pairs (capture turn, lookup turn, agent_notes-directed enrichment, refusal/clarify), replay post-refactor, verify equivalent semantics. **Cherry-picked from v0.3.0 #101** (composable prompt-section architecture). See `specs/customization-framework.md` § "Prompt decomposition contract" for full layer specification + implementation discipline.

3. ⏺️ **Fix Pro parent_slug over-emission on updates.** Pro emits `parent_slug` on `updates` entries when no hierarchy move was directed; destructive in the Social-Technology→top-level incident 2026-05-18. The prompt rule already says "only set parent_slug when explicitly directed" — it's not sticking. Two-part fix: (a) tighten + relocate the rule to higher prominence in §3 Routing under the new layered seam; (b) commit-side guard analogous to `detectNoOpUpdateFailure` — strip the field with a loud warning OR fail the ingestion when no hierarchy-move directive verb appears in the transcript. **Cherry-picked from v0.3.0 #102.** Natural fit with the prompt decomp work; behaves as a test case for the new layering.

4. ✴️ **Live Agent search-tool prompt cues — "when/how to search".** Prompt-side fix shipped 2026-05-19: new "never give up before exhausting search with reformulations" rule in `behavioral/tool-use.ts` + full-depth wiki tree in preload so Live Agent has every slug in context. Web-search reluctance also rebalanced (web is "the right tool, not a fallback" when the question is outside-the-wiki). Scalability ceiling documented in `tradeoffs.md` § "Full wiki tree in Live Agent preload". **Open (carried to v0.5.0):** preload-widening investigation (recency + project-graph proximity) for at-scale users. Beef up the §Tool palette section with concrete cues: when to search (entity not in preload, user references something by name that isn't in context, ambiguous referent across candidates), how to search (try explicit name first, fall back to semantic terms, chain `search_wiki` → `fetch_page` when a candidate looks promising). **Cherry-picked from v0.3.0 #100 (prompt-side slice).** Preload-widening slice carried to v0.5.0.

100. ✅ **Verbal tool-call acknowledgments — minor prompt patch.** Shipped 2026-05-19. Ordering rule explicitly requires the ack as the FIRST part of the response stream, BEFORE the function call, after dogfood revealed Gemini Live sometimes reorders parts so the ack landed AFTER tool completion (reading as backfilling). May need client-side investigation if prompt-strengthening alone doesn't fix the ordering — captured in commit message as a fallback path. Hardcoded Live Agent behavior (not a knob): briefly acknowledge when reaching for a tool ("looking that up...", "let me check...") before silent execution. Default on; no toggle. Transparency win — closes a UX gap where the user can't tell if the agent is thinking, searching, or stalled. Lands as part of the Track A1 prompt-decomposition pass (Behavioral layer adjustment). Source: 2026-05-19 KnobSpec workshop. *(Numbered 100 to avoid collision with B1 #5 — v0.3.0 used the same non-contiguous-renumbering convention.)*

## Demo target

End-of-sprint, a returning TestFlight user should notice:

1. Empty / "just testing" calls return instantly with no spurious processing noise.
2. Saying "be more terse with me" to Audri sticks across calls (NL overlay → knob distillation loop).
3. The Agents tile lists the default Assistant with editable persona + model-intelligence slider + dreaming toggle.
4. Dreams appear under the Assistant's detail view after a few days of use; tapping into one opens a "discuss next call" flow.
5. Notes pages cross-link to each other and show a `Related notes` footer.
6. Calendar subscriptions feed "what's on my plate this week" naturally.

If those six are true, v0.4.0 shipped.
